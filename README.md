# Data-Engineering-and-Business-Intelligence

This repository focuses on the **intersection of database architecture and visual storytelling**, showcasing the ability to transform raw data into actionable insights [User Query]. The projects included here demonstrate technical proficiency in **SQL database design**, **ETL processes**, and **advanced data visualization**, developed during my Bachelor of Science in Computer Science at Wilmington University, where I graduated **Summa Cum Laude** with a 3.93 GPA.

## 📊 Core Competencies
*   **Database Architecture:** Expert in relational database management systems (RDBMS) basics, table relationships, and implementing **3NF normalization** to ensure data integrity.
*   **Data Modeling:** Proficient in **Entity-Relationship (ER) modeling** using Chen and Crow’s Foot notation, as well as dimensional modeling involving facts and dimensions.
*   **Business Intelligence:** Advanced use of **Tableau** for big data visualization, creating integrated stories and chloropleth maps to analyze complex trends.
*   **Data Engineering:** Hands-on experience with **ETL (Extract, Transform, Load)** processes and data warehouse architecture.

## 📂 Repository Structure

### [Data-Warehousing](./Data-Warehousing)
Focuses on the structural components required for high-level data analysis and storage:
*   **Dimensional Modeling: Facts and Dimensions:** Detailed documentation on designing schemas for analytical workloads.
*   **ETL Process and Data Transformation Insights:** Discussion on the critical phases of the data development lifecycle and transformation logic.

### [Database-Design-SQL](./Database-Design-SQL)
Showcases the foundational engineering required to build robust, normalized data environments:
*   **ExoProtect Insurance Database Case Project:** A comprehensive case study in designing a database for the insurance industry.
*   **SQL Scripts for Funky Bizz Database:** Practical application of SQL for table creation, data insertion, and complex querying.
*   **Visual Table Normalization to 2NF and 3NF:** Exercises demonstrating the transition of unnormalized data into structured relational schemas.
*   **ER Modeling and Cardinality Constraints:** Technical design documents illustrating data relationships and constraints.

### [Visualizations-Tableau](./Visualizations-Tableau)
Features data-driven storytelling through visual analytics:
*   **Delaware Mortality Analysis:** Projects like the "Cloropleth Death Map - Delaware 2012-2014" utilize Tableau to visualize sensitive datasets, such as **opioid overdose trends**, to drive informed decision-making.
*   **Integrated Final Tableau Story:** A culmination of visual analysis techniques applied to complex datasets to present a cohesive narrative.
